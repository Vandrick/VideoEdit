VideoEdit release build notes
=============================

Use this when you are ready to create a release EXE manually.

1. Open PowerShell in the project folder:

   E:\projects\VideoEdit

2. Create the build venv if it does not already exist:

   C:\Users\gslas\AppData\Local\Programs\Python\Python314\python.exe -m venv .venv314

3. Install or refresh dependencies:

   .\.venv314\Scripts\python.exe -m pip install --upgrade pip setuptools wheel pyinstaller -r requirements.txt "rembg[cpu]" tbb

4. Build the EXE:

   .\.venv314\Scripts\python.exe -m PyInstaller --clean --noconfirm --workpath build_release --distpath dist_release VideoEdit.spec

5. The release EXE will be:

   dist_release\VideoEdit.exe

Important notes:

- VideoEdit.spec intentionally bundles rembg, onnxruntime, pymatting, and TBB.
- VideoEdit.spec intentionally excludes the root msvcp140.dll because that DLL conflicts with ONNX Runtime in the packaged app.
- If PyInstaller cannot replace an existing EXE, close any running VideoEdit.exe and try again, or build to a new dist folder.
- The first background-removal run may still download the selected rembg model into the user's model cache.
- ffmpeg.exe is expected to be available beside the app or on PATH if you are distributing export/open-video support.

Optional FFmpeg local install command:

Open PowerShell in the same folder as VideoEdit.exe, then paste:

   $ErrorActionPreference = "Stop"
   $zip = Join-Path (Get-Location) "ffmpeg-release-essentials.zip"
   $tmp = Join-Path (Get-Location) "ffmpeg_extract"
   Invoke-WebRequest "https://www.gyan.dev/ffmpeg/builds/ffmpeg-release-essentials.zip" -OutFile $zip
   Remove-Item $tmp -Recurse -Force -ErrorAction SilentlyContinue
   Expand-Archive $zip -DestinationPath $tmp -Force
   $bin = Get-ChildItem $tmp -Recurse -Filter ffmpeg.exe | Select-Object -First 1 -ExpandProperty DirectoryName
   Copy-Item (Join-Path $bin "ffmpeg.exe") (Join-Path (Get-Location) "ffmpeg.exe") -Force
   Copy-Item (Join-Path $bin "ffprobe.exe") (Join-Path (Get-Location) "ffprobe.exe") -Force
   Remove-Item $tmp -Recurse -Force
   Remove-Item $zip -Force
   .\ffmpeg.exe -version

The FFmpeg project links to gyan.dev for Windows EXE builds. The release essentials ZIP is enough for VideoEdit.

Quick smoke test:

   .\dist_release\VideoEdit.exe

Open a small image or video, delete a frame, copy/paste a frame, and try one background-removal pass before publishing.
